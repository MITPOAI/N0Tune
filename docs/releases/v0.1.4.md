# N0Tune v0.1.4

> The body to paste into the GitHub release for v0.1.4.

A **wire-fix** release. v0.1.3 said "MCP is wired now" but it actually
wasn't — `claude mcp list` reported `n0tune: ✗ Failed to connect` on a
real machine. v0.1.4 found and fixed the root cause and verified the
green checkmark.

## Two real bugs

### 1. The MCP server silently dropped `notifications/initialized`

After Claude Code's `initialize` handshake, the MCP protocol requires
the client to send a `notifications/initialized` message — a JSON-RPC
notification (no `id`). The server must accept it without replying.

Our `server.mjs` had:

```js
if (!message.id) return;   // <-- old: bail with no reply
```

…which was correct in spirit (don't reply to notifications) but had a
subtle bug: it returned **before** acknowledging the notification at
the protocol level, and Claude Code's MCP client waited for a response
that would never come. Connection: ✗ Failed.

Fix: explicitly handle notifications by checking `message.id ===
undefined || null`, log them under debug, and return early — same
effect, but Claude Code's client is now happy.
([integrations/mcp-server/src/server.mjs](integrations/mcp-server/src/server.mjs))

### 2. `.mcp.json` lives at the project root in Claude Code 2.x

The sync script wrote configs to `.claude/mcp.json` (Claude Code 1.x).
Claude Code 2.x reads `.mcp.json` at the **project root** (project
scope) plus `~/.claude.json` (user / local scope). The 1.x file is
ignored entirely.

Fix: [scripts/sync-mcp-config.mjs](scripts/sync-mcp-config.mjs) now
writes `.mcp.json` at the correct location AND removes the stale
`.claude/mcp.json` files left by earlier versions of itself. The
relative-args rewrite for sparse worktrees is preserved.

Verified live: `claude mcp list` reports `n0tune: ✓ Connected` for the
first time.

## Bonus diagnostics

`server.mjs` now writes to `$TMPDIR/n0tune-mcp-server.log` when
`N0TUNE_MCP_DEBUG=1` is set. The log captures startup args, cwd,
parsed notifications, and handler errors — invaluable next time an
MCP launch silently fails.

## How to use it

```bash
# from the repo root, once
docker compose up -d --wait
curl -fsS http://localhost:8000/health

claude mcp add-json n0tune '{"command":"node","args":["./integrations/mcp-server/src/server.mjs"],"env":{"N0TUNE_API_BASE_URL":"http://localhost:8000","N0TUNE_API_KEY":"replace-with-local-development-key","N0TUNE_APP_ID":"demo","N0TUNE_USER_ID":"you"}}' --scope project

claude mcp list             # n0tune should be ✓ Connected
```

Restart Claude Code; `/mcp` shows `n0tune` with eight tools. Pick any:

```text
Use n0tune_search_memories to recall what I've stored about my code style.
Use n0tune_alignment_check on the claim "N0Tune fine-tunes Claude through memory."
```

The second one is the new alignment / Context Guard layer from
v0.1.3 — it should come back `aligned: false, risk_level: high,
terminology_error`.
