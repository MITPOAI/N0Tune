# N0Tune v0.1.1

> This is the body to paste into the GitHub release for v0.1.1.

A polish release on top of [v0.1.0](v0.1.0.md). Same product surface, but
CI is now green, agents have a one-page operating manual, and the install
story is documented properly.

## What changed since v0.1.0

### CI is green

- **Gitleaks allowlist** — the v0.1.0 tag was tagged with a red CI
  because the generic-api-key rule flagged `X-N0Tune-API-Key` (our HTTP
  header *name*) in `CHANGELOG.md` prose. `.gitleaks.toml` now sets
  `regexTarget = "line"` and allowlists the header name + stopwords;
  `.gitleaksignore` pins the historical fingerprint.
- **`@n0tune/sdk` resolution** — `integrations/vercel-ai-sdk` and
  `apps/desktop` typecheck without flakes. `packages/sdk-js` got a
  `prepare` script (so `npm install` builds `dist/` automatically) and
  the Node CI job builds `@n0tune/sdk` before any workspace typechecks
  run.
- **Playwright e2e** — the dashboard's Refresh button had an
  `aria-label="Refresh data"` that shadowed the "Refresh" text, so the
  test's anchored `getByRole("button", { name: /^refresh$/i })` never
  matched. Removed the redundant aria-label. CI also now polls
  `http://localhost:3000` before kicking off Playwright, instead of
  hitting a half-booted Next.js server.

### Agent operating manual

- **`CLAUDE.md`** — read by Claude Code on every session start. Lists
  the armor framing, the hard rules (no hardcoded model, no iOS/Android,
  zero telemetry, local-first), the repo map, and the pre-PR check
  commands.
- **`AGENTS.md`** — the [agents.md](https://agents.md) standard, read by
  Codex CLI, Cursor, Aider, Continue, and any other coding agent. Same
  content, slightly tighter.

### Install doc

- **`docs/install.md`** — standalone install guide. Pre-built installer
  route for users, source-build route for devs, the "Run the Gateway"
  option, and an **exact paste-able prompt** that has Claude / Cursor /
  Codex CLI itself wire the MCP config for you.

### UI polish

- Light/dark mode in the Desktop app via `prefers-color-scheme`.
- Refined accent + warn palettes, focus rings (`0 0 0 3px accent-soft`),
  subtle motion on hover/focus, tabular numerals on the status overlay,
  sticky/blurred overlay surface.
- Dashboard `globals.css` + `tailwind.config.ts` get matched transitions,
  a soft-moss accent, surface/card/ring shadow tokens, and hover/active
  states on `.button` and `.input`.
- Dashboard header reframed from "Personal AI Runtime" to "Armor for your
  AI tools".

### Tauri cleanup

- Removed every `cfg(target_os = "android" | "ios")` gate from
  `apps/desktop/src-tauri/Cargo.toml` and `src/lib.rs`. The project
  targets Windows / macOS / Linux only; the mobile conditionals were
  dead weight.
- `tauri-plugin-global-shortcut` is now an unconditional dependency.

### Dogfooding

- `scripts/seed-dogfooding.ps1` now indexes the armor docs alongside
  `context-compiler.md`:
  `product-direction.md`, `how-it-works.md`, `install.md`,
  `wire-to-claude.md`, `wire-to-codex-cli.md`, `wire-to-gemini-cli.md`,
  `CLAUDE.md`, `AGENTS.md`.
- Seed memories explicitly state the armor framing, the
  no-hardcoded-model rule, zero-telemetry promise, and
  Windows/macOS/Linux-only target.
- Three context previews run end-to-end ("how does N0Tune compile
  context?", "what is N0Tune and how is it different from a chat app?",
  "how do I wire N0Tune to Claude Code?") so an MCP-wired client comes
  back framed correctly.

## Anything else?

Nothing functional changed — same MCP tools, same provider router, same
context compiler, same continual-learning loop, same Tauri + SQLite +
keychain stack as v0.1.0. This release is about making the CI badge
honest, helping future agents stay on-frame, and polishing the surfaces
the user actually looks at.

For the full feature surface, see [`v0.1.0.md`](v0.1.0.md).
