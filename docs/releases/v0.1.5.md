# N0Tune v0.1.5

> The body to paste into the GitHub release for v0.1.5.

A "make it actually useful" release. Unifies the design language,
adds a dashboard view that shows how the system thinks, makes the
token savings unmissable, proves N0Tune is dogfooded with a
verifiable evidence log, and brings the rate limiter up to the 2026
standard with proper headers. Compiler retrieval gets its first
dedicated unit-test file.

## Why this one isn't just "another context compressor"

Five additions that, together, separate N0Tune from "memory plugin":

1. **Visible system reasoning.** The new dashboard *Live trace* tab
   draws every step of one request — memory bars ranked by retrieval
   score, MMR drops with citations, token math, cache check, pipeline
   strip. You can watch N0Tune think instead of trusting it.
2. **Adaptive persona** (`POST /v1/users/{user_id}/style/adapt`).
   Reads the last 20 preference memories and proposes tone / depth /
   format flips when ≥ 40 % of them agree. Deterministic, no LLM.
   The persona auto-tunes to what the user has actually been saying.
3. **Token savings in the user's face**. New `SavingsHero` on Desktop
   Home shows three tabular-numeric cards: tokens saved this session,
   percentage vs naive baseline, average per request. Hidden when
   you haven't chatted yet (no dishonest counters).
4. **Verifiable dogfooding**. New `docs/dogfood-evidence.md` is a
   live log of memory IDs consulted while this PR was written. Six
   memories saved + queried back from the running Gateway during
   v0.1.5 work; you can reproduce the queries from your own
   `docker compose up` and see them.
5. **Rate-limit headers**. Every `/v1/` response now carries
   `X-RateLimit-Limit / -Remaining / -Reset`; 429 also sets
   `Retry-After`. The sliding-window in-memory backend and the
   fixed-window Redis backend both return a single typed
   `RateLimitDecision` so the middleware doesn't branch.

## Design language

The Desktop palette wins. The dashboard adopts the same CSS-variable
tokens (`--bg / --surface / --line / --ink / --accent / --warn` …)
and gains a proper `@media (prefers-color-scheme: dark)` block —
previously dashboard dark mode was broken. Desktop dark-mode line
contrast was bumped from `#2e2a25` (1.6:1) to `#3d3930` so cards
read clearly. Tailwind theme is bound to the CSS variables so
classnames stay short.

## Compiler

- New `apps/api/app/tests/test_compiler_internals.py` with 7 unit
  tests covering `_diversify_memories`: keeps distinct memories,
  drops near-duplicates citing the canonical id, filters zero-score
  rows, keeps when below threshold, skips rows missing embeddings,
  threshold inclusivity, preserves score order among kept.
- This closes the audit gap: the MMR + state-weight code paths now
  have direct test coverage so threshold changes break loudly.
