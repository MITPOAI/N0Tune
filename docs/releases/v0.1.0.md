# N0Tune v0.1.0

> This is the body to paste into the GitHub release for v0.1.0.

## What is N0Tune?

**Armor for the AI tools you already use.** Local memory, token-savings,
and tailored context for Claude Code, Claude Desktop, Cursor, Codex CLI,
Gemini CLI, ChatGPT, and any OpenAI-compatible client.

Same model. Same question. Personalized answer. Fewer tokens. No fine-tuning.

The point: don't open another chat app. Add N0Tune as armor *around* the
tools you already love.

## What ships in v0.1.0

### Wires into your existing AI tool

| You're using…                                | How it integrates                                                              |
| -------------------------------------------- | ------------------------------------------------------------------------------ |
| Claude Code / Claude Desktop / Cursor        | MCP (seven tools). [`docs/wire-to-claude.md`](../wire-to-claude.md).           |
| Codex CLI                                    | MCP. [`docs/wire-to-codex-cli.md`](../wire-to-codex-cli.md).                   |
| Gemini CLI                                   | `n0tune compile` adapter + tray hotkey. [`docs/wire-to-gemini-cli.md`](../wire-to-gemini-cli.md). |
| ChatGPT / any OpenAI client                  | Gateway proxy at `/v1/openai/chat/completions`.                                |
| Anything else (editor, browser, terminal)    | Desktop tray + global hotkey captures clipboard → memory.                     |

### Desktop (Tauri)

- Tray icon + global hotkey (`Cmd+Shift+Space` / `Alt+Space` default).
- Quick-remember overlay (clipboard pre-fill, ⌘+Enter to save).
- Live status overlay: tokens used this session, cache hit rate, active
  memory count, tailored-vs-naive prompt ratio, last provider used.
- Window close hides to tray instead of quitting — N0Tune stays ambient.
- Onboarding, chat (real provider calls via fetch), memory viewer,
  context preview, persona settings, provider settings.
- SQLite storage and OS-native keychain for API keys (macOS Keychain,
  Windows Credential Manager, Linux Secret Service / kwallet).

### Gateway (FastAPI)

- Context Compiler with prompt-injection scoring, semantic cache,
  memory lifecycle (state + decay), memory scopes, RBAC, audit logs,
  rate limiting, SSE streaming proxy.
- Continual-learning loop: `POST /v1/memories/consolidate` clusters
  similar memories and collapses them into a denser summary.
- Three provider wire shapes:
  - `openai` — OpenAI, OpenRouter, Ollama, LM Studio, vLLM, anything
    OpenAI-compatible.
  - `anthropic` — Anthropic Messages API.
  - `gemini` — Google `generateContent` REST.

### Integrations

- `@n0tune/sdk` (TypeScript) and `n0tune` (Python) SDKs.
- `n0tune-langchain` — `BaseRetriever` + memory helper.
- `n0tune-llamaindex` — `BaseRetriever` returning `NodeWithScore`.
- `@n0tune/vercel-ai-sdk` — provider factory.
- `n0tune-markdown-folder` — first connector.
- MCP server with seven tools.

### CLI

`n0tune` Node binary. Subcommands:

- `doctor`, `demo`, `compile <message>`, `memory list/add/delete/export/consolidate`,
  `persona export/import`, `files sync`, `mcp install`.

Reproducible against an honest "stuff everything in" baseline using the
deterministic hash embedding backend. Higher with real embeddings.

## Download

We don't ship signed installers yet — **you build them**.

### Prerequisites

- Node 20+, npm.
- Rust toolchain via [rustup](https://rustup.rs/).
- OS-specific build tools per
  [v2.tauri.app/start/prerequisites](https://v2.tauri.app/start/prerequisites/).

### Build

```bash
git clone https://github.com/n0tune/n0tune.git
cd n0tune
npm install

# One-time: generate platform-specific icon formats from the source PNG.
npm --workspace apps/desktop run tauri:icon

# Build installers.
npm --workspace apps/desktop run tauri:build
```

Output paths under `apps/desktop/src-tauri/target/release/bundle/`:

| OS       | File                                                              |
| -------- | ----------------------------------------------------------------- |
| Windows  | `nsis/N0Tune_0.1.0_x64-setup.exe` or `msi/N0Tune_0.1.0_x64_en-US.msi` |
| macOS    | `dmg/N0Tune_0.1.0_aarch64.dmg` (or `_x64` on Intel)               |
| Linux    | `appimage/n0tune_0.1.0_amd64.AppImage` or `deb/n0tune_0.1.0_amd64.deb` |

macOS users: bundles aren't notarized. After install, one-time:

```bash
xattr -dr com.apple.quarantine /Applications/N0Tune.app
```

Or right-click → Open the first time to bypass Gatekeeper.

## Run

Desktop:

```bash
# After installing
open -a N0Tune                 # macOS
"C:\Program Files\N0Tune\N0Tune.exe"   # Windows
n0tune                         # Linux (AppImage / deb)
```

The tray icon shows a hammer mark. Click it for menu items, or press
`Cmd+Shift+Space` / `Alt+Space` to quick-save a memory from anywhere.

Gateway (optional — needed for MCP / OpenAI proxy / multi-device sync):

```bash
docker compose up -d --wait
curl http://localhost:8000/health
```

Wire to Claude / Cursor / Codex CLI:

```bash
node packages/cli/bin/n0tune.mjs mcp install
# Paste the printed JSON into your client's MCP config.
```

## What's deliberately not in v0.1.0

These are explicit follow-ups:

- **Signed installers** — you build them. The hosted release pipeline
  lands once we have code-signing certificates.
- **Latency-true streaming pass-through** — current SSE fans out a
  resolved answer; upstream pass-through is a follow-up.
- **Floating widget** — design doc only (`docs/floating-widget.md`).
  The tray icon + global hotkey are the v0.1.0 alternative.
- **Real evals beyond token_savings** — directories and methodology
  are documented; only `token_savings_eval` is fully implemented.
- **MCP-native Gemini CLI** — Gemini CLI's MCP support is still
  evolving; v0.1.0 ships an adapter (`n0tune compile`) and the tray
  hotkey path instead.

The "honest status board" in [`docs/product-direction.md`](../product-direction.md#honest-status-board)
is the canonical "what works right now" table. If anything in this file
disagrees with it, the table wins.
