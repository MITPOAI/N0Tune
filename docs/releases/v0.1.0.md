## What is N0Tune?

**N0Tune turns any AI model into your personal AI — without fine-tuning.**

Bring GPT, Claude, Gemini, Qwen, OpenRouter, Ollama, LM Studio, or any
OpenAI-compatible model. N0Tune adds local memory, style, files, semantic
cache, and a context compiler on top.

The model is unchanged. The **prompt** is compiled per request from your
local memory + style + files. That pattern is what we call **context-tuning**
(see [context-tuning.md](../context-tuning.md)).

## What's in v0.1.0

### Five surfaces, all wired

| Edition           | What                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| **Gateway**       | FastAPI server. Memory, RAG, semantic cache, OpenAI-compatible proxy with SSE streaming, RBAC, audit logs, rate limiting, prompt-injection scoring. Mature. |
| **Desktop**       | Tauri + React app. Onboarding, chat (real provider calls), memory viewer, context preview, persona, provider settings. **You build the installer from source**; signed binaries land in a later release. |
| **Core**          | `packages/core` Python lib — the extracted context compiler + interfaces both Gateway and (eventually) Desktop's Rust side consume. |
| **CLI** (`n0tune`)| Node binary: `doctor`, `demo`, `memory`, `persona`, `files`, `mcp install`, plus honest "coming next" stubs for `init` / `desktop start` / `gateway start`. |
| **MCP**           | Stdio server with seven tools (`n0tune_search_memories`, `_save_memory`, `_get_style_profile`, `_search_docs`, `_context_preview`, `_forget_memory`, `_get_persona`). Wires N0Tune memory into Claude Desktop / Claude Code / Cursor. |

### Provider router

Three wire shapes, selected by `N0TUNE_PROVIDER_KIND`:

- **`openai`** — OpenAI / OpenRouter / Ollama / LM Studio / vLLM / any
  OpenAI-compatible endpoint.
- **`anthropic`** — Anthropic Messages API (`/v1/messages`).
- **`gemini`** — Google Gemini `generateContent` REST.

The Desktop renderer calls these directly from TypeScript using the user's
key (stored in `localStorage` today, OS keychain when the Rust layer
lands).

## Quick start

```bash
git clone https://github.com/n0tune/n0tune.git
cd n0tune
cp .env.example .env

# Gateway (developer/server mode)
docker compose up -d --wait
curl http://localhost:8000/health

# Desktop dev shell (web preview, no Tauri needed)
npm install
npm --workspace apps/desktop run dev
# open http://localhost:1420

# Native Tauri build (Rust + OS toolchain required)
npm --workspace apps/desktop run tauri:icon   # once
npm --workspace apps/desktop run tauri:build
# installers under apps/desktop/src-tauri/target/release/bundle/
```

## Wire it to Claude

```bash
node packages/cli/bin/n0tune.mjs mcp install
# Paste the printed JSON into your Claude Desktop / Claude Code config.
```

Full walkthrough in [wire-to-claude.md](../wire-to-claude.md). Includes
the terminal smoke-test you can run without an LLM client.

## What's deliberately not shipped yet

These are explicit follow-ups, not promises we missed:

- **Signed Desktop binaries** — you build them. The hosted release
  pipeline lands once the Rust SQLite layer is in (Phase G/J).
- **SQLite + OS keychain inside Tauri** — Today the renderer persists to
  `localStorage`. The Rust side has one `runtime_info` command.
- **Floating widget / tray icon** — design doc only
  (`docs/floating-widget.md`).
- **Real evals beyond token_savings** — the directories and methodology
  are documented; only `token_savings_eval` is fully implemented.
- **Latency-true streaming pass-through** — current SSE fans out a
  resolved answer; upstream pass-through is a follow-up.
